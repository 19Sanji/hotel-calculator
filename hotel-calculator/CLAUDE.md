# Бухта Кила — Калькулятор стоимости

Веб-приложение для расчёта стоимости проживания в отеле «Бухта Кила».
Воспроизводит логику ценообразования из Excel-таблицы с поддержкой нескольких
расчётов в одном заказе и реестром сохранённых заказов.

## Стек

- **React 18** + **TypeScript** (strict)
- **Vite** — сборка и dev-сервер
- **Ant Design v6** — UI-компоненты
- **date-fns** — работа с датами
- **dayjs** — форматирование дат в Ant Design DatePicker
- **GitHub Pages** — деплой (base: `/hotel-calculator/`)

## Команды

```bash
npm install        # установить зависимости
npm run dev        # запустить dev-сервер (http://localhost:5173)
npm run build      # TypeScript-проверка + production-сборка
npm run preview    # предпросмотр production-сборки
```

## Архитектура состояния

Всё состояние живёт в `App.tsx`:

| Переменная      | Тип           | Хранение      | Описание |
|----------------|---------------|---------------|---------|
| `currentOrder` | `Order`       | память        | Текущий рабочий заказ в калькуляторе |
| `savedOrders`  | `Order[]`     | память сессии | Сохранённые заказы (реестр) |
| `roomTypes`    | `RoomType[]`  | localStorage  | Тарифная сетка |

`savedOrders` **не сохраняются** между перезагрузками страницы — только `roomTypes`.

## Типы номеров (RoomType)

Поля, хранящиеся в localStorage и редактируемые в «Справочнике цен»:

| Поле           | Описание |
|----------------|---------|
| `priceWeekday` | Цена за будничную ночь (пн–чт) |
| `priceWeekend1`| Цена за выходную ночь отдельно |
| `priceWeekend2`| Цена за пару пт+сб (за 2 ночи) |
| `adultWeekday` | Надбавка за доп. взрослого в будни |
| `adultWeekend` | Надбавка за доп. взрослого в выходные |
| `childRate`    | Надбавка за ребёнка за ночь |
| `maxAdults`    | Макс. доп. взрослых (0 = запрещено) |
| `maxChildren`  | Макс. доп. детей (0 = запрещено) |

## Логика расчёта

Файл: `src/lib/calculator.ts`

1. Подсчёт ночей: будничные, выходные, пары пт+сб (`dateUtils.ts`)
2. Базовая стоимость номера по тарифной сетке
3. Надбавки за доп. взрослых и детей
4. Расчёт всех применимых скидок:
   - Раннее бронирование (выезд − бронирование ≥ 30 дней → −10%)
   - Именинник (−15%)
   - 3+1 / 4+1 (≥3 или ≥4 ночи пн–чт подряд → одна ночь бесплатно)
   - Длительный заезд (≥7 ночей → все ночи по будничному тарифу)
   - Ручная скидка (любой процент от 0 до 100)
5. Автовыбор лучшей скидки по максимальной сумме; при равенстве — по приоритету Excel H3

`allDiscounts[0]` **всегда** «Нет акции» (amount=0) — это важный инвариант,
который используют `ResultDisplay` и `format.getPanelTotal`.

## Форма и ограничения по доп. местам

`CalculatorForm` использует `Form.useWatch('roomTypeId')` для реактивного
отслеживания выбранного типа номера. При смене типа:
- поля «Доп. взрослых» и «Доп. детей» зажимаются до `maxAdults`/`maxChildren`;
- если `maxAdults === 0` — поле дизейблится и обнуляется.

## Ключевые решения

| Проблема | Решение |
|----------|---------|
| React 18 StrictMode дважды вызывал `setOrders` в `editSavedOrder` | Убрали вложенный setState, читаем `savedOrders` напрямую из замыкания |
| `useEffect` для заполнения формы при редактировании | `key={panel.id}` на `CalculatorForm` — перемонтирует форму при смене панели |
| Дублирование `fmt` и `getPanelTotal` в 4 компонентах | Вынесено в `src/lib/format.ts` |
| Счётчики `orderCounter`/`panelCounter` сбрасывались при HMR | Заменены на `crypto.randomUUID()` |
